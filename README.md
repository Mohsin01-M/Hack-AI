# BCU AI Hackathon 2026 – Participant Repo

## Challenge
Build a Generative AI question-answering pipeline that answers **100 multiple-choice questions**.

You may use prompting, internet-assisted search, embeddings, retrieval, ranking, RAG, or any other AI pipeline, provided your selected LLM is **not larger than 8B parameters**.

The goal is to maximise answer accuracy while providing a reproducible and well-documented AI solution.

## Time and team rules
- Duration: **4 hours**
- Team size: **1–3 members**
- Model limit: **maximum 8B parameters**
- Internet-assisted answering is allowed
- Submit your answer file, code, and PowerPoint in your own GitHub repository
- Submit the GitHub link using the Google Form or Microsoft Form shared in Microsoft Teams

## Files
| File | Purpose |

## Required submission
Your GitHub repository must contain:

1. Your final answer file, named `TEAMNAME_submission.csv` **Please rename the file with your team name.**
2. Your code, including any notebooks/scripts used to generate the answers
3. A PowerPoint presentation, preferably PDF or PPTX
4. A short README explaining how to run your solution
5. Submit your GitHub repository link to Dr Nouh Elmitwally via the submission form or email (Nouh.Elmitwally@bcu.ac.uk).

## Answer format
Your final answer file must be named:

```text
TEAMNAME_submission.csv
```

It must contain exactly these columns:

```csv
question_no,answer
1,A
2,C
3,B
```

Allowed answers are `A`, `B`, `C`, `D`, `E`, or `Unknown`.

## Suggested development workflow
1. Load `questions_100.csv`.
2. For each question, create a search query from the question and options.
3. Retrieve useful evidence from the internet or your chosen knowledge source.
4. Rank retrieved snippets using embeddings, TF-IDF, reranking, or another method.
5. Build a RAG-style prompt containing the question, options, and top-ranked evidence.
6. Ask your chosen LLM to return only one letter: `A`, `B`, `C`, `D`, `E`, or `Unknown`.
7. Validate the output format and export `TEAMNAME_submission.csv`.
8. Push your code, answer file, and presentation to GitHub.

## Model rule
You may choose any model, open-source or closed-source, but the model must not exceed **8B parameters**. In your slides/README, clearly state the model name and size.

## Important
Do not submit the organiser answer key. Your repo should only contain your own generated answers and method.
